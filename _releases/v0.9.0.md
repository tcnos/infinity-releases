A large batch spanning everything merged since v0.8.0 — new navigation, list ergonomics, and a major round of legacy-data migration (profile attachments, note replies, company geography), plus storage and consistency fixes.

## Features

### [INVOS-39] Fund slug in every URL
Every fund-scoped screen now carries the fund's slug in its URL (`/:slug/portfolio`, `/:slug/investors`, …). URLs are now shareable, bookmarkable, and unambiguous when you work across multiple funds — a link always opens the right fund's context instead of whichever was last active.

### [INVOS-27] Company logos across all list views
Portfolio, Dealflow, Sourcing and Archive lists now show each company's logo beside its name. Scanning a long list is far faster with a visual identifier instead of text alone.

### [INVOS-29] Sortable company list columns
Company lists are now sortable by any column (name, sector, portfolio date, stage, …), server-side and consistent across every page of results — not just the rows currently loaded. Sorting is case-insensitive.

### [INVOS-77] Sortable investor list columns
The same server-side, all-pages column sorting now applies to the investor list.

### [INVOS-34] Legacy company profile attachments
Profile attachments uploaded in the legacy application now appear in the new app under **Profile → Attachments**. Files are pulled from the legacy GridFS store and land in each company's drive `profile/` folder, alongside the logo.

### [INVOS-86] Company Country & State from legacy
Company Profile now shows **Country** and **State/Province**, resolved from the legacy location master during the data pull. Previously these were blank because the source only carried a flat address string.

---

## Enhancements

### [INVOS-78] Modular backend routes
The monolithic `inf.routes.js` has been split into domain-scoped routers. No user-facing change — this is a maintainability improvement that makes the backend safer and faster to extend.

---

## Fixes

### [INVOS-87] Unified logo storage location
Migrated logos were written under a divergent storage folder (`company/`) while everything else used `companies/`, producing two folders for one scope. Logos always served correctly, but the store is now consolidated to a single `companies/` folder, with a migration that relocates existing logos safely.

### [INVOS-85] Migrated logos now preview inline
Migrated company logos opened as downloads instead of previewing in the browser, because the source returned a generic content-type and the preview flag was unset. Logos now carry a real image mime-type and preview inline.

### [INVOS-43] Legacy timeline note replies migrated
Replies to legacy timeline notes were being dropped during migration (they live as separate `comment` records, not under the note). They are now migrated into Discussions as threaded replies, with author and timestamp preserved.

### [INVOS-80] Breadcrumbs carry the fund slug
A regression from the URL restructuring left some breadcrumb links without the fund slug, breaking navigation back up the hierarchy. Breadcrumbs now prefix the active fund slug correctly.

### [INVOS-74] Industry managed list seeds on sync
The INDUSTRY managed list is now seeded during the Mongo→PG sync, not only via a one-off migration, so it's present after a fresh data pull.
