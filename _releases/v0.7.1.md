## Features

### [INVOS-19] Company timeline history ported to Discussions
The historical company timeline from the legacy platform — investment notes, member feedback, deal updates — is now available in the new **Discussions** tab, with the original authors and dates preserved. File attachments from those notes are carried across too: each is stored in the company's Drive under a `timeline/` folder and linked inline in the discussion. Authors who didn't yet exist in the new system are created automatically so attributions stay intact. This brings years of deal context into the platform where the team already works.

---

## Fixes

### [INVOS-24] PDFs and images downloaded instead of opening in a tab
Clicking a PDF or image attached to a discussion (or in the Drive tab) forced a download instead of opening it for a quick look. Browser-viewable files — PDFs, images, video, audio and text — now open inline in a new tab, while other file types continue to download cleanly. Applies across every storage backend.

### [INVOS-36] Editing a post broke its attached file reference
When editing a discussion post or comment that contained a Drive file link, the filename became editable as plain text and the link could be silently broken on save. File links are now protected while editing — they stay a solid, intact reference you can remove but not accidentally corrupt.

### [INVOS-31] Could not delete a discussion thread that had replies
Deleting a parent discussion that had replies failed silently with a console error. The replies are now removed along with the parent, and the delete confirmation tells you how many replies will be affected before you confirm.

### Editing or deleting a migrated discussion returned "Forbidden" (403)
Discussions ported from the legacy timeline could not be edited or deleted — even by an admin — because they lacked the fund association the permission check required. Permissions now resolve correctly for admins and fund editors, and the affected records have been corrected.

## Deployment notes

No schema migrations in this release.

**To port the company timeline into Discussions (INVOS-19)**, run the migration on the server once. It requires VPN access to the legacy database; pass the connection string via the environment (never hardcode it):

```bash
docker compose exec \
  -e LEGACY_DB_URI="$LEGACY_DB_URI" \
  inf-backend sh -c "cd /app && node tools/migrate-timeline-discussions.js --with-files"
```

The migration is idempotent and safe to re-run. Drop `--with-files` to port text only (Phase I); attachments still in legacy MinIO/Google/S3 (not GridFS) are skipped and logged.
