## Features

### [INVOS-101] Email Tagging — Emails tab + Chrome extension
Deal correspondence now lives alongside the rest of the deal. A view-only **Emails** tab on each company shows tagged emails with their attachments, and a **Chrome extension** tags an email (or a whole thread) to a company directly from web Gmail/Outlook — so the deal record finally captures the conversation, not just the numbers.

### [INVOS-98] Report Builder — self-serve custom reporting
Fund managers can build their own reports without waiting on engineering — pick the data, shape the columns, and export. Removes the bottleneck of one-off report requests.

### [INVOS-96] Application Form Enhancement — OTP, drafts, captcha
The public application form is now a proper two-step flow: applicants enter name/email first, verify by **email OTP** (verified mode) or pass a **self-hosted captcha** (open mode), then fill the form. They can **save a draft and resume later** (form-configurable expiry), and a refresh-safe session keeps them in place. Fund managers get an Inbox **Drafts** tab showing who started but hasn't submitted.

### [INVOS-111] Deal pull / push / lifecycle across funds
Manage a deal's presence across funds safely: **pull** (co-evaluate), **push** (move to another fund with bucket+stage), and a context-aware **remove** (delink when in >1 fund, else soft-delete to a 30-day **Trash** with Restore). A hard **financial guard** blocks removing any deal that has money in a fund — server-side, not just hidden.

### [INVOS-83] Configurable deal cards + stage aggregates
Deal cards are now configurable (choose which fields surface), and pipeline stages show live aggregates — a faster, more tailored dealflow view.

## Enhancements

### [INVOS-112] Custom fields → global admin catalog
Custom fields are now defined **once** at the admin level (a global catalog, like managed lists) with **company-level shared values**, instead of fragmented per-fund definitions where the same label could silently not share data. Form-submitted custom-field values now flow through to the company's Custom Fields tab.

### [INVOS-108] Multi-fund Mongo→PostgreSQL sync
The data pipeline was rebuilt as a true multi-fund flow — companies, users, stages, and financials sync per-fund from the authoritative source, so a deal shared across funds is modelled correctly (one company, N fund links).

### [INVOS-106] Legacy tagged-email + attachment import
Historic tagged emails and their attachments are ported from the legacy store into the new Emails tab, so the email history is there from day one — not just newly-tagged mail.

### [INVOS-126] Scoped migration tooling
`./upgrade.sh` is now flexible for ops: scope a load to specific company **buckets** (portfolio/dealflow/sourcing/…) and/or a **sample size**, opt into which **file categories** are pulled, layer buckets on incrementally with **`--add`** (no re-sync, no data loss), with **live progress** and **concurrent** file transfers to S3.

## Fixes

### [INVOS-110] Company discussions scoped per fund
On a deal shared across funds (one company, multiple fund links), discussions were leaking between funds — a note posted under one fund appeared under another. Discussions are now correctly scoped per (company, fund), and stay dormant/reattach as a deal moves between funds.
