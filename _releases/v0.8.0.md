12 tickets ship in this release — the new Industry managed list, two report fixes, and a set of data-integrity, cache, and UX fixes (including a P0 on user editing).

## Features

### [INVOS-26] Custom managed list for Industry
A company's Industry is now a centrally-managed list instead of a free-text field. The field is a searchable dropdown with type-to-filter and inline "quick-add", so teams pick from a consistent vocabulary and new sectors are added on the fly without leaving the company page. The list is seeded from the industries already present on your companies, so nothing is lost on upgrade.

---

## Enhancements

### [INVOS-71] Managed list "Replace & Remove" migrates referencing records
When you remove a value from a managed list and choose a replacement, every record that referenced the old value is now repointed to the new one — no more orphaned values left behind on companies. The replacement is stored as its canonical display value, and any open company view refreshes to show the migrated value.

### [INVOS-65] Move a company to a stage in an adjacent pipeline bucket
The company stage dropdown now offers stages from adjacent buckets (Sourcing → Dealflow → Portfolio), so you can advance a company across the pipeline directly from its profile instead of being limited to its current bucket.

### [INVOS-69] Fund scope surfaced in global search
Global search now shows which fund each result belongs to, making it easier to find the right company or investor when the same name exists across funds.

### [INVOS-50] Cancel button to revert unsaved edits
The investor edit view now has a Cancel action that discards unsaved changes and restores the original values, so an accidental edit no longer forces a manual undo.

### [INVOS-44] Attach a new file when posting or replying
You can now upload a brand-new file directly while composing a post or reply via the /file picker, instead of only being able to reference files that were already uploaded.

## Fixes

### [INVOS-73] Admin > Users: editing a user blanked their name (P0)
Editing any user in Admin > Users and saving wiped their first and last name. The first/last name split had never been backfilled onto existing users, so the edit form opened blank and the save overwrote the name with empty values. Fixed by backfilling names, refusing to overwrite a name with blanks, pre-populating the form, and enforcing the mandatory first-name rule on save.

### [INVOS-72] App-wide staleness — pages showed old values after changes
After making a change, some pages continued to show the previous values until a manual refresh. Cache freshness was rebalanced across the app so data updates are reflected on navigation without the jank of force-refetching every screen.

### [INVOS-66] Transaction report "Type" showed EQUITY/DEBT instead of the security
The Transactions report labelled rows with the broad asset class (EQUITY/DEBT) rather than the actual instrument. It now shows the specific security (e.g. CCPS, CCD), matching how the investment is actually structured.

### [INVOS-53] Rollup report was not working
The rollup report returned no usable rows. The endpoint was rebuilt to return consolidated, all-investor rows with the correct sign convention, so the report renders again.

### [INVOS-49] Deleting a pipeline stage dropped its companies
Deleting a stage that still held companies removed those companies from the board in the frontend. Deleting a non-empty stage is now blocked, and any companies previously orphaned by a stage deletion are recovered.

### [INVOS-42] Only the Admin user appeared as a Fund Manager
The Fund Manager dropdown under Admin > Funds listed only the admin account. It now lists all users, and persists and reloads the selected manager correctly.

## Deployment notes

**Run incremental migrations after pulling new images — no data re-seed required:**
```bash
docker compose exec inf-backend sh -c "cd /app && npx knex migrate:latest"
docker compose restart inf-backend
```

Migrations applied:
- `015_industry_lookup_seed` — seeds the INDUSTRY managed list from existing company sectors (INVOS-26)
- `016_backfill_user_names` — backfills first/last name from the existing display name for all users (INVOS-73)
