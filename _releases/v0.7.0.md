## Features

### [INVOS-22] Investor Entity Merge
Investors often hold investments under multiple entities (individual, HUF, trust, etc.) that need to be consolidated. A new **Merge** action on the Investor Profile lets you transfer all investments, balances, and fund memberships from one entity to another and delete the source — keeping the investor's history intact under a single entity. The merge is protected by a conflict check: if both entities already have positions in the same funding round, the merge is blocked to prevent data integrity issues.

## Enhancements

### [INVOS-17] First Name + Last Name — Investors & Users
The single full-name field has been split into separate **First Name** and **Last Name** fields across both Investors and Users. This makes names structured and searchable, and enables proper personalisation in outbound emails. Existing records are automatically backfilled by splitting on the first space. All add and edit forms now show the two fields side by side; the legacy combined name column is kept in sync for backward compatibility with any integrations reading it.

### [INVOS-18] Company Address — Country and State
The Company Profile now includes structured **Country** (dropdown, ISO-code backed) and **State / Province** (free text) fields alongside the existing address block. Country uses the same lookup as the Investor Profile so values are consistent across the platform. Both fields are optional and stored without requiring a data re-seed.

## Fixes

### [INVOS-23] Portfolio Manager Not Saving on Company Edit
Editing a company's Portfolio Manager and saving appeared to succeed but the change was silently discarded. The PUT payload was missing the `fund_id` context that the backend required to locate the correct PM record. Fixed — PM assignments now persist correctly on save.

### [INVOS-21] Admin Users List — Alphabetical Sort
The Users list in the Admin panel was ordered by creation date (newest first), making it hard to find specific users as the list grew. It now sorts alphabetically by name by default.
