## What's new

### PM / Fund-team user import (sync-inf step 8)
- New `seed-inf-team-users.js` — imports Portfolio Managers and fund-team members from legacy `teams` + `users` collections into `inf_users` (role: `user`) and `inf_fund_users` (role: `editor`). Idempotent upsert — never overwrites existing user passwords or roles.
- `sync-from-remote` now pulls the full legacy `users` collection (~27.9k docs) with sensitive fields excluded (`enterprise_token`, `temp_enterprise_token`, `verification_code`, `createPassword`, `ip`, `host`, `last_login_ip`).
- Wired as **step 8** of `sync-inf` — runs automatically on full sync.

### ME toggle fix (Portfolio & Dealflow)
- The **Me | All** filter on Portfolio and Dealflow pages was silently ignored by the backend. Fixed: `pm_id` is now extracted from `req.query` and applied as a `WHERE fund_company_team.user_id = pm_id` filter on both the main query and the count query.

## Upgrade notes
- Re-run `sync-from-remote` (to pull `users`) then `sync-inf` (step 8 will populate `inf_users` + `inf_fund_users`), then `migrate-mongo-to-pg --only=users,fund_users` to push to PostgreSQL.
- No schema changes — no migration required.
